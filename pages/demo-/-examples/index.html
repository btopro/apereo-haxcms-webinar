We'll start with three unique examples (that we've all done).

  • A slide player . This site is a testbed but illustrates a simple back and forth navigational pattern.
  • A blog website , my own blog in fact. This illustrates a simple medium clone, complete with rss/atom feeds.
  • A documentation website . HAXTheWeb.org is powered by HAXcms, leveraging HAXeditor in order to edit and maintain the content.
  • For bonus, this website your viewing currently is HAXcms powered and is an alternative documentation / course content theme.

All four websites are powered by the exact same data structure and editing environment. They follow the same publishing workflow (to github pages) and can have their themes radically changed without it affecting content. Their content (via web components) can have accessibility radically improved after publishing without touching the end-point material.
